# План учёта времени (15‑мин единицы)

## Требования
- Перерыв не учитывается: фиксируется только как pause/resume.
- Единица учёта: 15 минут.
- Округление: по правилу порога 10 минут (>=10 → вверх, иначе вниз).
- Время локальное.
- Слоты в отчёте объединяются (не показываем «тупо по 15 минут»).
- Хранение: в корне проекта.
- Авто‑подстановка ветки/шага: да.
- Таймер start/stop: обязателен.

## 1) Хранение данных
- Файл `time_log.md` в корне проекта.
- Формат событийного лога:
  ```
  2026-02-04 10:00 | event: start | activity: coding | branch: A_main | step: A_002 | note: правки UI
  2026-02-04 10:35 | event: pause | note: перерыв
  2026-02-04 10:50 | event: resume
  2026-02-04 11:05 | event: switch | activity: reading | note: read README
  2026-02-04 11:20 | event: stop | note: закончил
  ```
- Активности: `bb`, `reading`, `coding`.

## 2) Авто‑подстановка ветки/шага
- При `start/switch/resume`:
  - если есть git‑ветка шага (например `A_002`) → `branch=A`, `step=A_002`;
  - иначе берём последний шаг из `branches/<branch>/runs/`;
  - если ничего нет — записываем пусто, но событие создаём.

## 3) Учёт времени (без перерывов)
- Рабочие интервалы: `start/resume` → `pause/stop/switch`.
- Перерывы (`pause` → `resume`) исключаются из учёта.

### Округление до 15 минут
- Берём остаток минут от ближайшего предыдущего 15‑мин рубежа.
- Если остаток **>= 10** → округляем вверх.
- Иначе → округляем вниз.

## 4) Объединение слотов
- В отчёте объединяем **соседние** интервалы с одинаковыми:
  - activity + branch + step + note.
- Если note отличается — блоки остаются отдельными.

## 5) Отчёт (детальный «что делал»)
- Сводка по активности (мин/ч).
- Сводка по веткам/шагам.
- Детальные блоки с диапазоном времени + note.
- Если note пустой, дополняем из:
  - `journal.md`,
  - `prompt.md`/`evaluation.md`,
  - git log (последний коммит в диапазоне).

## 6) CLI команды
- `time start <project_path> --activity bb|reading|coding [--note "..."]`
- `time pause <project_path> [--note "..."]`
- `time resume <project_path> [--note "..."]`
- `time switch <project_path> --activity ... [--note "..."]`
- `time stop <project_path> [--note "..."]`
- `time report <project_path> [--date YYYY-MM-DD | --week YYYY-WW | --range ...]`

## 7) Веб‑интерфейс (после CLI)
- Страница “Time log” с кнопками Start/Pause/Resume/Stop, активностью и заметкой.
- Авто‑подстановка branch/step.
- Кнопка “Generate report” с выводом/скачиванием отчёта.

## 8) Тесты
- Корректность pause/resume.
- Округление по правилу >=10.
- Объединение слотов.
- Агрегация по активности/веткам/шагам.
