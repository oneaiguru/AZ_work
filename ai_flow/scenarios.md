# Основные сценарии использования AI Flow

Каждый сценарий содержит ключевые команды для сборки проекта «с нуля» и примерный текст промпта, который вы сохраняете в `prompt.md` шага.

1. **Инициализация нового проекта и первой ветки**
   - **Что делаем**: создаём директорию проекта, запускаем `init-project` и фиксируем стартовую ветку `A/001`.
   - **Команды**:
     ```bash
     python ai_flow.py init-project ./my-ai-project --title "Исследование RAG-подхода" --date 2026-02-04
     python ai_flow.py create-branch ./my-ai-project A_main --title "Базовая стратегия"
     ```
   - **Промпт**:
     ```text
     Цель: понять, как собрать и оценить RAG-цепочку для внутреннего справочника. Контекст: есть каталог документов и простой поисковый слой.
     Задача: описать архитектуру потока обработки, критерии успеха и вопросы, которые нужно прояснить перед первым экспериментом.
     ```

2. **Пошаговое развитие основной ветки**
   - **Что делаем**: добавляем последовательные шаги в ветку `A_main`. Каждый шаг создаёт prompt/context/result_raw/evaluation и коммитит.
   - **Команды**:
     ```bash
     python ai_flow.py new-step ./my-ai-project A_main A_001
     python ai_flow.py new-step ./my-ai-project A_main A_002 --from-step A_001
     ```
   - **Промпт для A_001**:
     ```text
     Объясни, какие источники данных нужны, как их индексировать и как построить поиск по релевантности.
     Укажи метрики (recall/precision) для первой итерации и предложи короткий plan для их измерения.
     ```
  - **Промпт для A_002**:
    ```text
    Основываясь на результатах A_001, предложи финальный prompt для генерации ответов из найденных фрагментов.
    Опиши ожидаемый формат результата, примеры и способы проверки качества.
    ```
  - **Пример результата (result_raw.md)**:
    ```text
    🔍 Источники: базы знаний sales/tech.csv + internal/faq.md.
    📌 Ответ формируется по шаблону.
    ✅ Проверка: сравнение с золотыми ответами (MAP@5 = 0.68, recall ~0.75). Промежуточный вывод: подмешивание эмбеддингов помогает точности, но снижает coverage.
    ```
  - **Журнал**: добавьте строку в `journal.md`, например
    ```
    2026-02-04 | A_main/A_002 | status: partial | Итог: сформирован prompt + метрики, нужен шаг с фильтрацией источников.
    ```
  - **План**: обновите `plan.md`, отметив активную ветку `A_main`, следующую задачу типа `Исследовать фильтрацию источников (A_003)`.

3. **Создание альтернативной ветки**
   - **Что делаем**: ветка `B` от `A_002` исследует иную гипотезу, сохраняет привязку к шагу `from_step`.
   - **Команды**:
     ```bash
     python ai_flow.py create-branch ./my-ai-project B_alt-from-A_002 --parent A_main --from-step A_002 --title "Генерация без постфильтра"
     python ai_flow.py new-step ./my-ai-project B_alt-from-A_002 B_001 --from-step A_002
     ```
  - **Промпт**:
    ```text
    Цель: сравнить генерацию ответов без постфильтрации с полнотекстовым поиском. Дай пошаговый алгоритм обработки запроса, укажи, как использовать from_step в контексте сравнения.
    ```
  - **Пример результата и связанный prompt**: если в `result_raw.md` появился текст с силой справки и метрикой `precision@3`, зафиксируйте это в `journal.md` и `plan.md`, чтобы затем связать шаг `B_001` с будущими `C_*`.

4. **Оценка и фиксация результатов шага**
   - **Что делаем**: после каждого шага обновляем `evaluation.md`, можно добавлять тесты рядом с шагом.
   - **Промпт**:
     ```text
     Подведи итоги шага: что получилось, какие метрики и ссылки на результаты (result_raw). Выставь статус success/partial/fail и объясни, что нужно сделать дальше.
     ```
   - **Тест**: файл `branches/B_alt-from-A_002/runs/B_001/check_result.py` может проверять текст `result_raw.md`, запуская `python -m unittest`.

5. **Закрытие ветки и генерация диаграммы**
   - **Что делаем**: закрываем ветку, фиксируем статус и сохраняем диаграмму в `branches/diagram.mmd`.
   - **Команды**:
     ```bash
     # редактируем branches/B_alt-from-A_002/branch-info.md и ставим status: success
     python ai_flow.py diagram ./my-ai-project --output branches/diagram.mmd
     ```
   - **Промпт** (для финального шага):
     ```text
     Напиши короткое резюме всей ветки: гипотеза, результаты, финальные метрики. Предложи следующие шаги или причину закрытия.
     ```

6. **Повторное использование шаблонов и переход к новой ветке**
   - **Что делаем**: при переходе на новый эксперимент создаём новый шаг с `--from-step` либо новую ветку, используя результаты предыдущих шагов.
   - **Команды**:
     ```bash
     python ai_flow.py create-branch ./my-ai-project C_next --from-step B_001
     python ai_flow.py new-step ./my-ai-project C_next C_001 --from-step B_001
     ```
   - **Промпт**:
     ```text
     Используй вывод шага B_001 как контекст. Опиши изменённую стратегию генерации, укажи, какие ограничения сохраняются, и как проверять успешность.
     ```

В каждом сценарии сохраняйте промпт в `branches/<branch>/runs/<step>/prompt.md`, добавляйте `context.md` с необходимыми деталями и фиксируйте `result_raw.md` + `evaluation.md`. Рекомендуется по окончании каждого шага запускать `git status` и коммитить изменения.
