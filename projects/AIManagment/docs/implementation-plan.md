# AIManagment — план реализации

## Шаг 1. Документы и постановка

**Цель:** описать продукт, MVP и этапы реализации.

Результат:
- зафиксирован scope;
- определены ключевые сущности;
- описаны ближайшие шаги разработки.

## Шаг 2. Статический MVP

**Цель:** собрать первый рабочий интерфейс без backend.

Задачи:
- создать дашборд с ключевыми метриками;
- добавить форму создания AI-зачи;
- вывести канбан-представление по статусам;
- реализовать фильтрацию и поиск;
- сохранить данные в `localStorage`.

Критерий готовности:
- пользователь может открыть страницу и полностью работать локально.

## Шаг 3. Улучшение UX

**Цель:** сделать прототип удобнее для ежедневного использования.

Задачи:
- добавить редактирование карточек;
- улучшить адаптивность;
- добавить быстрые действия для смены статуса и приоритета;
- продумать пустые состояния и подсказки.

## Шаг 4. Данные и интеграции

**Цель:** подготовить проект к реальному использованию командой.

Задачи:
- определить API-контракты;
- подготовить backend-модель;
- спроектировать хранение пользователей, задач и истории изменений;
- оценить интеграции с внешними системами.

## Текущий фокус

Шаг 4 завершён на уровне проектирования. Следующий логичный этап — реализация
backend API, справочников пользователей/моделей и серверной истории изменений.
